## **Amazon S3**

- Amazon S3는 AWS의 주요 구성 요소 중 하나이며, "무한하게 확장할 수 있는" 스토리지입니다.
- 많은 웹 사이트가 Amazon S3에 의존하고 있으며, 많은 AWS 서비스들도 Amazon S3를 통합하여 사용합니다.
- 정적 웹사이트 호스팅 기능도 제공합니다.
    - 403 Forbidden 에러가 발생하면, 버킷 정책이 공개 읽기를 허용하는지 확인해야 합니다.
- 백업 및 스토리지, 재해 복구, 아카이브, 정적 웹사이트 호스팅, 미디어 호스팅 등..

### Bucket

- 파일을 버킷에 저장합니다.
- 버킷은 전체 지역과 모든 계정에서 전역적으로 고유한 이름을 가져야 합니다.
    - AWS에서 전역적으로 고유한 단 하나의 이름이여야 합니다.
- 리전 수준에서 정의합니다.

### Objects

- 파일은 키를 가지고 있으며, 키는 파일을 가지고 있는 경로를 의미합니다.
- 접두사와 객체 이름으로 구성됩니다.
- 객체의 최대 크기는 5TB이며, 매우 큰 파일을 업로드할 경우에는 multi-part upload를 사용해야 합니다.

### Security

**User-Based**

- **IAM Policeis**: IAM에서 특정 사용자에 대해 허용되어야 하는 API 호출 지정

**Resource-based**

- **Bucket Policies**: S3 콘솔에서 직접 할당할 수 있는 전체 버킷 규칙
    - 다른 계정의 사용자 허용 = S3 버킷에 액세서할 수 있는 교차 계정 (cross account)
- **Object Access Control List (ACL)**: 더 세분화된 권한 설정 (비활성화 가능)
- **Bucket Access Control List (ACL)**: 덜 일반적인 권한 설정 (비활성화 가능)

**Encrytion**

- 암호화 키를 사용하여 Amazon S3 객체를 암호화

### **S3 Bucket Policies**

```jsx
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicRead",
            "Effect": "Allow",
            "Principal": "*",
            "Action": [
	            "s3:GetObject"
            ],
            "Resource": [
	            "arn:aws:s3:::bucket/*"
            ]    
    ]
}
```

### Replication (CRR & SRR)

- 소스 버킷과 복제 대상 버킷에서 둘 모두 버전 관리 기능이 활성화되어야 합니다.
- **Cross-Region Replication (CRR)**: 교차 리전 복제
- **Same-Region Replication (SRR)**: 같은 리전으로 복제
- 버킷은 다른 AWS 계정에 속할 수 있습니다.
- 복제는 비동기적으로 진행됨 - 복제 과정은 백그라운드에서 이루어지게 됩니다.
- S3에 적절한 IAM 권한, 즉 읽기 및 쓰기 권한을 부여해야합니다.
- 사용 사례:
    - **CRR**: 법규, 내부 체제 관리, 데이터가 다른 리전에 있어 발생할 수 있는 지연 시간 줄이기, 계정 간 복제
    - **SRR**: 로그 통합, 운영 환경과 개발 환경간의 실시간 복제

### S3 Stroage Classes

<img width="951" height="377" alt="Image" src="https://github.com/user-attachments/assets/fac55888-76f9-4061-853a-cf815d1b1b40" />

**S3 Standard – General Purpose**

- 자주 액세스하는 데이터에 사용되며, 지연 시간이  짧고 처리량이 높습니다.
- ex) 빅데이터 분석, 모바일 및 게임 애플리케이션, 콘텐츠 배포

**S3 Storage Classes – Infrequent Access**

- 덜 빈번하게 액세스되지만 필요한 경우 빠른 액세스가 필요한 데이터에 사용됩니다.
- S3 Standard보다 저렴하지만, 검색 비용이 발생합니다.

**Amazon S3 Glacier Storage Classes**

- 아카이빙 및 백업용으로 사용되는 저비용  객체 스토리지
- 가격: 스토리지 비용 + 객체 검색 비용
- ex) Instant(밀리초), Flexible(분), Deep Archive(시간)

**S3 Intelligent-Tiering**

- 소액의 월간 모니터링 및 자동 계층화 비용이 발생합니다.
- 사용량에 기반하여 객체를 자동으로 액세스 계층 간에 이동합니다.
